## 1. 产品概述

### 1.1 一句话定位
一款面向知识类视频博主的 iOS 录制工具，**一次拍摄同时产出横屏（16:9）与竖屏（9:16）两个独立视频文件**，免去同一段内容反复重录或后期重剪的成本。

### 1.2 产品名（暂定）
**CoFrame**（Co-Frame，双构图同录）

### 1.3 解决的核心痛点
知识博主（口播 / 教学 / 测评 / 访谈）通常需要把同一段内容同时分发到：
- **横屏渠道**：YouTube、B 站主站、视频号横版、播客视频版
- **竖屏渠道**：抖音、小红书、Instagram Reels、视频号竖版、TikTok

当前工作流的痛点：
1. 重录两遍：表达不一致、时间翻倍。
2. 录一遍横屏后裁竖屏：人物主体常被切掉，需要后期 AI 重构图，不可控。
3. 双机位拍摄：成本高、需要双设备同步、对一人创作者门槛高。

CoFrame 的策略：**单一物理镜头 + 单次拍摄 + 双路异构输出**，让博主在拍摄时就能同时看到"横屏构图"与"竖屏构图"是否都成立。

## 2. 目标用户与典型场景

### 2.1 目标用户画像
- **核心用户**：独立知识博主 / 中小型 MCN 创作者，1 人或 2-3 人小团队
- **设备**：iPhone 12 及以上（用户自有设备拍摄，对画质有要求）
- **使用场景**：固定机位口播、桌面演示、户外短 vlog、对谈采访
- **技术水平**：会用剪映 / Final Cut，但不写代码；希望"录完即得双版本"

### 2.2 典型使用流程（用户故事）
1. 我打开 App，自动横屏，看到一个大的横屏取景框，**右上角或一侧浮着一个小的竖屏取景框**（实时同步）。
2. 我调整手机机位，确认人物在横屏构图里居中、在竖屏构图里也没有被切到关键部分。
3. 我打开"九宫格基准线"对齐镜头，关掉，开始录制。
4. 录完点停止，App **不会自动写入相册**，而是把两段视频存到 App 内的"草稿箱"。
5. 我在草稿箱里回看横屏 / 竖屏两个版本，删掉拍砸的，把满意的两个版本分别"导出到相册"或"分享"。

## 3. 关键需求分析与技术取舍

> 这一节是 PRD 的"心脏"。下面 3 个决策直接决定了产品的形态与开发成本，请优先确认。

### 3.1 ✅ 决策 #1 — 双路视频如何"同时录制"

**最终方案：A — 单镜头 + 双路裁切**

iPhone 的硬件 / iOS API 现实：**同一个物理摄像头无法在硬件层面同时输出两路独立分辨率/方向的视频流**。可选实现方案有三种：

| 方案 | 实现方式 | 优点 | 缺点 |
|---|---|---|---|
| **A. 单镜头 + 双路裁切（推荐）** | 用后置主摄以最高分辨率 4K 横屏采集，`AVAssetWriter` 同时写出两个文件：① 完整 16:9 横屏；② 从横屏中心（或用户指定区域）裁出的 9:16 竖屏 | 稳定、画面一致、功耗低、所有 iPhone 12+ 都支持 | 竖屏视野是横屏的子集，最大宽度受限；人物离镜头太近时竖屏会切到肩膀 |
| **B. 多摄像头同录** | `AVCaptureMultiCamSession` 同时启用主摄 + 超广角，主摄出横屏、超广角出竖屏 | 两路视野独立、竖屏可以更广 | 仅 iPhone XS+ 支持；两个镜头色彩 / 畸变不一致后期难统一；功耗发热显著 |
| **C. 单镜头 + 用户可调裁切框** | 在方案 A 基础上，竖屏裁切框允许用户在横屏画面里**拖动定位 + 录制中跟踪**（如人物追踪） | 构图自由度最高 | 录制中实时人物追踪算法复杂，v1 风险高 |

v1 采用方案 A；"裁切框可拖动定位（不在录制中追踪）"作为 v1.1 扩展点。

### 3.2 ✅ 决策 #2 — 竖屏取景框的视觉位置

**最终方案：a — 悬浮 PiP**

"打开 App 即横屏，同时看到竖屏取景框" 有两种主流交互：

- **方案 a · 画中画悬浮**：横屏全屏预览作为主画面，竖屏构图以一个半透明矩形框 + 缩略实时预览悬浮在右上 / 右下角（可拖动、可隐藏）。更接近 iPhone 原生相机的"克制"感。
- **方案 b · 双框并列**：左侧大横屏预览，右侧贴边并列一个真实尺寸的竖屏预览。信息更"全"，但占屏面积大、横屏构图被压缩。

v1 采用悬浮 PiP，最大化保护横屏取景体验，符合"主+辅"视觉层级。

### 3.3 ✅ 决策 #3 — 录制画质与文件大小

**最终方案：3 档 H.264 — 1080p30 / 4K30 / 4K60**

iPhone 主摄常用录制规格与单分钟体积参考：

| 规格 | 分辨率 | 帧率 | 码率参考 | 1 分钟体积（横屏 H.264） |
|---|---|---|---|---|
| 高清 HD | 1920×1080 | 30fps | ~8 Mbps | ~60 MB |
| 4K 标准 | 3840×2160 | 30fps | ~50 Mbps | ~370 MB |
| 4K 高码率 | 3840×2160 | 60fps | ~100 Mbps | ~750 MB |

**双路同录 ≈ 横屏体积 + 竖屏体积**（竖屏是裁切后重编码，约为横屏 30~40% 体积）。

v1 编码统一 H.264，确保最大兼容性；HEVC / ProRes 不在 v1 范围内。

### 3.4 ✅ 决策 #4 — 镜头切换

**最终方案：v1 支持前后摄像头切换**（后置主摄 + 前置自拍）。超广角与变焦镜头延后。

### 3.5 ✅ 决策 #5 — 音频

**最终方案：自动选用系统默认音频输入，双路视频共用同一份音频；不做监听耳返。**

## 4. 功能需求（FRD）

### 4.1 录制主界面（横屏 Landscape）

#### 4.1.1 取景预览
- **F-1.1** 横屏主预览：全屏（或近全屏）实时预览，AVCaptureVideoPreviewLayer，目标延迟 < 80ms。
- **F-1.2** 竖屏副预览（PiP）：悬浮在主预览右上角，默认尺寸约 90×160pt，半透明白色描边，内容与主预览实时同步，反映"录制后竖屏视频会得到的画面"。
- **F-1.3** PiP 可拖动至四角；可双击隐藏 / 调出。
- **F-1.4** PiP 内显示当前裁切区域在横屏中的位置（即在横屏预览上叠加一个对应的 9:16 取景框线）。

#### 4.1.2 基准线 / 构图辅助
- **F-2.1** 基准线类型 v1 支持：
  - **九宫格**（rule of thirds，最常用）
  - **中心十字**
  - **水平仪**（基于 CMMotionManager 的实时倾角，倾斜超阈值变红）
- **F-2.2** 通过顶部菜单一键切换：关 / 九宫格 / 十字 / 水平仪 / 全部。
- **F-2.3** 基准线**同时显示在横屏主预览和 PiP 竖屏预览上**（相对于各自构图比例计算）。
- **F-2.4** 基准线只是 UI 叠加层，**绝不烧录进视频**。

#### 4.1.3 录制控制
- **F-3.1** 中心红色录制按钮（参考系统相机），点击开始 / 再次点击结束。
- **F-3.2** 录制中：按钮变方形红，顶部显示红点 + 计时（HH:MM:SS）。
- **F-3.3** 录制中禁用：分辨率切换、镜头切换、PiP 拖动锁定（可选）。
- **F-3.4** 录制中支持锁屏吗？**建议 v1 不支持**——录制时屏幕保持唤醒（`UIApplication.isIdleTimerDisabled = true`）。
- **F-3.5** 异常中断（电话、低电量、内存压力）：自动停止录制，已录片段写入草稿箱并标记"中断"。

#### 4.1.4 相机参数控制（参考系统相机的克制设计）
- **F-4.1** 对焦：单击预览任意位置触发对焦 + 测光，黄色方框动效。
- **F-4.2** 曝光锁定：长按预览触发 AE/AF 锁。
- **F-4.3** 曝光补偿：对焦框出现后，旁边出现"小太阳"滑条，上下拖动 ±2EV。
- **F-4.4** 缩放：双指捏合，1× ~ 数字变焦（v1 不做光学切换）。
- **F-4.5** 闪光灯（Torch）：v1 只在录像时支持持续开 / 关。
- **F-4.6** **不做** v1：手动白平衡 / ISO / 快门 / Log 模式（专业相机 App 范畴，超出范围）。

#### 4.1.5 顶部 / 侧边工具栏（横屏布局）
- 左侧竖排工具：基准线开关 / 闪光灯 / 镜头切换（前后）
- 右侧竖排工具：分辨率档位 / 设置入口 / 进入草稿箱
- 录制按钮居中于右侧（横握姿势下右手拇指顺手）

#### 4.1.6 强制方向
- **F-6.1** App 全程强制 `landscape`（左右两个方向均可），不响应竖握自动旋转。
- **F-6.2** 设备方向指示（陀螺仪检测到设备实际竖握）：在角落给一个图标提示"请横握"，但不阻塞功能。

### 4.2 草稿箱（Drafts）

#### 4.2.1 入口与列表
- **F-7.1** 录制主界面右上角"胶卷"图标 → 进入草稿箱。
- **F-7.2** 列表按拍摄时间倒序，每条录制为一个"录制会话"卡片，包含：
  - 一对缩略图（横屏 + 竖屏）
  - 时长、文件大小、分辨率档位、拍摄时间
  - 状态徽章（正常 / 中断 / 已导出过）

#### 4.2.2 单条详情与回看
- **F-8.1** 点击卡片进入详情：顶部 Tab 切换 [横屏 ↔ 竖屏]，AVPlayer 播放。
- **F-8.2** 操作按钮：
  - **导出到相册** —— 可选 [仅横屏 / 仅竖屏 / 两个都导]
  - **分享** —— 调系统 Share Sheet（Files、AirDrop、第三方 App）
  - **删除** —— 删除整个会话或单一方向版本
  - **重命名**
- **F-8.3** 导出到相册需通过 `PHPhotoLibrary` 申请"添加照片"权限（注意：iOS 14+ 推荐使用 `addOnly` 权限，最小化用户授权范围）。

#### 4.2.3 存储管理
- **F-9.1** 文件存储位置：App 沙盒 `Documents/Recordings/<sessionId>/landscape.mov` & `portrait.mov` & `meta.json`。
- **F-9.2** 草稿箱顶部展示：已用空间 / 设备剩余空间。
- **F-9.3** 剩余空间 < 1GB 时录制前弹出警告。
- **F-9.4** 一键清理已导出过的草稿（带二次确认）。

### 4.3 设置页

- 默认分辨率档位
- 默认基准线类型
- 录制时自动静音系统提示音（开 / 关）
- 关于 / 隐私政策 / 反馈邮件

## 5. 非功能需求

### 5.1 性能
- 启动到取景框可见 ≤ 1.5s（冷启动）
- 预览延迟 ≤ 80ms（人眼几乎无感）
- 4K30 双路录制时 CPU 占用 ≤ 60%、机身温度 30 分钟内不触发系统降频
- 录制 30 分钟内不崩溃、不丢帧（关键 KPI）

### 5.2 兼容性
- iOS 17.0+
- iPhone 12 及以上（A14 起）
- 横屏布局适配 iPhone 12 mini ~ iPhone 16 Pro Max（含动态岛安全区）

### 5.3 隐私
- 摄像头权限：录制必需
- 麦克风权限：录制必需
- 相册"仅添加"权限：仅在用户首次点击"导出到相册"时申请
- 不收集任何用户数据，不接入任何第三方 SDK（v1）

### 5.4 可访问性（v1 最小集）
- 关键按钮 VoiceOver 标签齐全
- 录制按钮震动反馈（开始 / 结束 Haptic）

## 7. UI / UX 设计原则

参考 iPhone 原生相机 App，要做到：

1. **克制**：默认界面只露出"取景 + 录制按钮 + 模式标签"，所有高级控件在用户主动触发时浮现。
2. **熟悉**：手势（点击对焦、长按锁定、捏合缩放）与系统相机一致。
3. **专业感**：黑色底 + 高对比度白色 / 黄色 + 红色录制色，无多余装饰。
4. **横屏原生**：所有按钮的可达性按"双手横握"姿势设计，常用操作集中在右侧拇指区。

具体视觉稿建议进入 v0.2 阶段由设计师产出（或可用 design:design-critique 技能辅助评审）。

## 8. 技术架构建议（供工程评估）

### 8.1 关键技术栈
- **语言**：Swift 5.10+
- **UI**：SwiftUI 主体 + UIKit 桥接相机预览层（`UIViewRepresentable` 包 `AVCaptureVideoPreviewLayer`）
- **相机**：`AVFoundation` —— `AVCaptureSession` + `AVCaptureVideoDataOutput`
- **写文件**：两个 `AVAssetWriter` 实例，共用同一份 `CMSampleBuffer`：
  - Writer A：原始横屏 buffer 直写
  - Writer B：用 `vImage` 或 `CIImage` 把横屏 buffer 中心裁切为 9:16，再写入
- **音频**：单一 `AVCaptureAudioDataOutput`，sample buffer 同时投递给两个 Writer
- **存储**：`FileManager` Documents 目录；元数据用 `Codable` 写入 `meta.json`
- **草稿箱回看**：`AVPlayer` + SwiftUI `VideoPlayer`
- **导出相册**：`PHPhotoLibrary.shared().performChanges` + `addOnly` 权限
- **传感器**：`CoreMotion` (`CMMotionManager`) 实现水平仪

### 8.2 关键技术风险
| 风险 | 严重度 | 缓解 |
|---|---|---|
| 双路写入帧率掉到 24fps 以下 | 高 | 第一个里程碑就做性能基准测试；裁切走 GPU（Metal / `CIContext.render(to: CVPixelBuffer)`）而非 CPU |
| 长时间录制发热降频 | 中 | 监听 `ProcessInfo.thermalState`，过热时自动降档并提示 |
| 录制中后台 / 来电 | 中 | 监听 `AVCaptureSessionWasInterrupted`，正确收尾 Writer，避免文件损坏 |
| 沙盒文件膨胀 | 中 | 草稿箱有空间提示 + 一键清理；导出后引导用户主动删除 |

### 8.3 工程脚手架建议（首版目录）
```
CoFrame/
  App/                  # SwiftUI 入口、AppDelegate
  Features/
    Capture/            # 录制主界面（VM + View + 相机管线）
    Drafts/             # 草稿箱列表 + 详情
    Settings/           # 设置
  Core/
    Camera/             # AVCaptureSession 管理
    Recording/          # 双路 AssetWriter 管线
    Storage/            # 沙盒文件管理 + meta.json
    Export/             # PHPhotoLibrary 封装
    Motion/             # CoreMotion 水平仪
  UI/                   # 共享组件（基准线层、PiP 容器、录制按钮）
  Resources/            # Assets / Localizable.strings
```
